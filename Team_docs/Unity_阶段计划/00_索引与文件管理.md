# Unity 阶段计划 — 索引与文件管理

> 本目录集中存放「星穹铁道」Unity 实现的**分阶段执行计划**，供不同任务执行者按顺序或按模块执行。  
> 依据：[技术文档_星穹铁道_Unity.md](../技术文档_星穹铁道_Unity.md)、[PRD_星穹铁道.md](../PRD_星穹铁道.md)。

---

## 1. 文件管理约定

### 1.1 本目录（Team_docs/Unity_阶段计划/）

| 约定 | 说明 |
|------|------|
| **命名** | `NN_简短标题.md`，NN 为两位数字序号（00、01、02…），便于排序与引用。 |
| **索引** | 本文件（00_索引与文件管理.md）为入口，列出全部阶段、文件名、目的与依赖。 |
| **新增阶段** | 新增阶段文档时，在本文件「阶段清单」中增加一行，并注明上一阶段编号（依赖）。 |
| **不修改上游** | 阶段计划仅引用技术文档与 PRD，**不修改**技术文档与 PRD 内容。 |

### 1.2 与 Assets 的对应关系

- 阶段计划描述**在 Unity 内要做的事**（场景、Prefab、脚本、配置、美术等）。
- 实际资源与脚本落在 **Assets/Game/** 下，目录结构见 [01_资源管理与文件布局.md](./01_资源管理与文件布局.md)。

## 3. 执行顺序建议

1. **先做 01**：确保 `Assets/Game` 及子目录已存在。
2. **02 与 03**：可并行或先 02 再 03（先有场景再在场景中摆 Prefab 占位）。
3. **04 及以后**：在场景与 Prefab 就绪后，按脚本 → 数据对接顺序推进。
