# Star Express 项目目录规范（Unity）

本文档用于约束后续资源放置位置，避免目录重复和路径混乱。

## 1. 当前目录（重构后）

```
Assets/
├── Game/
│   ├── 场景/
│   ├── 脚本/
│   ├── 预制体/
│   ├── 配置/
│   ├── 美术/
│   ├── UI/
│   ├── 编辑器/
│   ├── Resources/
│   └── 归档/
├── UI Toolkit/            # Unity 默认运行时主题资源
└── UOSLauncherEncrypt/    # 第三方/外部目录
```

## 2. 放置约定

- 正式业务资源统一放在 `Assets/Game/` 下。
- 场景只放在 `Assets/Game/场景/`（不再使用其他 `Scene/Scenes` 目录）。
- 脚本只放在 `Assets/Game/脚本/`。
- 预制体只放在 `Assets/Game/预制体/`。
- ScriptableObject 配置只放在 `Assets/Game/配置/`。
- 游戏 UI（UXML/USS）放在 `Assets/Game/UI/`。
- 美术资源放在 `Assets/Game/美术/`。
- `Assets/Game/归档/` 仅用于历史版本，不参与日常开发。

## 3. 命名规则

- 采用“英文根目录 + 中文业务目录”混合方式，便于团队理解和 Unity 习惯兼容。
- 同类资源禁止多目录并存（例如不能同时出现 `场景` 与 `Scenes` 作为正式目录）。
- 历史版本统一进入 `归档`，并加 `旧版` 标识，避免与正式资源同级混放。

## 4. 本次已完成整理项

- 将 `_Project` 迁移为 `Game`，并按中文业务目录重排。
- 消除重复场景目录：正式场景统一到 `Game/场景`。
- 消除重复资源目录：脚本、预制体、配置、美术、UI 均统一入口。
- 旧版场景、旧版预制体、冲突 UI 文件迁入 `Game/归档`。
- 代码中的编辑器路径常量已更新为 `Assets/Game/...`。

## 5. 后续建议

- 在 Unity Editor 打开项目后等待一次完整导入。
- 检查 `Build Settings`，仅保留 `Assets/Game/场景/` 下需要打包的场景。
- 若确认 `归档` 目录中的资源不再需要，可分批删除。
